# Session 2: the D5 matrix day — embedding pin, pass-weight knee, text-channel verdict

## Done

All full-epoch arms (113K steps, b=256, v3 conditioning, 600-batch final eval, pilot store), serial on the GPU while d6ext generated underneath at nice-19:

- **Embedding ablation → [ADR-0007](../decisions/ADR-0007-embedding-pin.md): statistical tie, pin stays Qwen3.** run5-bge vs run4: honest 0.9630 vs 0.9636, nonpass 0.8350 vs 0.8373, target identical to 4 decimals, value identical. BGE-M3 = validated drop-in escape hatch (m1-plan risk line resolved). Caveats recorded: in-pool + rung-1 only — cross-card generalization structurally invisible to this eval.
- **Pass-weight sweep (runs 7/4/6 = 1.0/0.3/0.1): knee at 0.3.** 1.0→0.3 buys +7.3pp nonpass for −0.4pp honest (nearly free); 0.3→0.1 another +3.7pp for −1.3pp (3× the price). Targets/X/value flat across the whole sweep — the knob is fully localized to the pass boundary. **Default set to 0.1** (action-rich prior for M2: passivity is the worse RL failure mode; the honest cost is the recalibratable part — weighted CE ≈ prior shift, one PASS-logit offset at inference re-dials the action rate post-hoc). 0.3 recorded as the knee if honest ever gets squeezed. Final adjudication belongs to D8 eval-path play (agreement can't rank the extra actions against a weak heuristic expert).
- **Null-text arm (run8, `--null-text`, pw=0.3 vs run4): rung-1 barely reads text.** Zeroing the entire text buffer: honest −0.19pp (borderline ~2.8σ), everything else within noise. So the ablation tie means "text barely used," not "both encoders equally good"; encoder-scale questions (8B) closed for rung-1 in-pool; **shortcut learning confirmed** — with ID available the text pathway doesn't get built. ID dropout = the lever to force it alive pre-Tutor (ADR-0007 amended).
