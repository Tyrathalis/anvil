# Session 2 (evening): fork-fidelity harness built; measurement pipeline running overnight

## Done

- **Fork-fidelity differential test designed, built, validated, and running** (ADR-0001 M0 deliverable (b)). Design note: [fork-fidelity-test.md](../design/fork-fidelity-test.md). Implementation is `forge forkcheck` in the fork (commits `d68d3ed`, `8f6f09e`): forks a live game via `GameCopier` at a quiescent active-player MAIN1 priority point, replays the fork to completion with a serialized clone of the `MyRandom` state, diffs per-turn state digests (life/counters/mana + per-zone card multisets, library **in order**) against the mainline, then compares outcomes.
- **Archaeology findings that shaped the harness:** `GameEventTurnBegan` fires mid-phase-advance (fork there and the copy never untaps); `GameCopier` doesn't copy the waiting-trigger queue (drain triggers before forking, defer if the stack is non-empty); a fresh copy's `PhaseHandler` silently *skips priority for the forked phase* — fixed with a 1-method engine patch (`devResumeAtPriority()`), which is the first concrete brick of the "single-step fork API" upstream story. Thread-interrupting timeouts replaced with watchdog draw-clocks so a hung fork can't leak a thread that keeps consuming the shared RNG singleton.
- **Detector validated end-to-end** with a `-perturb` mode (1-life delta injected into the fork): 3/3 detected at exactly the first turn boundary after the fork, diff sample pinpointing the life line.
- **First fidelity numbers (run 1 in flight, v1 semantics, n=41 at time of writing):** ~37% clean (fork reproduces mainline trajectory *and* outcome exactly), **~56% trajectory divergence** (typically manifesting turns later as library-order drift — i.e., an earlier decision delta desyncing the RNG stream), **~7% static mismatch** = the copy differs at copy time. Two concrete `GameCopier` bug classes already isolated, both seed-reproducible: **face-down/hidden exiled cards come back face-up in the copy** (seeds 20260719, 20260741; the PRUNE_HIDDEN_INFO TODO made flesh), and **a duplicated "Keyword Effects" object in the Command zone** (seed 20260723). Copy cost is a non-issue: 7–30 ms.
- Fork commits: `d68d3ed` (forkcheck v1 + `devResumeAtPriority`), `8f6f09e` (v2: fork only at active-player priority windows; `-freshrng` discriminator).

## Broke / surprised me

- Smoke test came back 3/3 clean and looked like GameCopier vindication; 41 games later the divergence rate is ~56%. Small-n optimism, the old story.
- The heuristic AI consults `MyRandom` at 82 call sites — so *any* single decision delta between mainline and fork silently desyncs the RNG stream, and the first visible symptom (library order after a shuffle) can be many turns downstream of the actual divergence point. Trajectory comparison catches this; single-point state comparison (all upstream has today) never would.

- **Bottleneck decomposition** (vmstat + live clocks + GC logs, w=1→16 per-worker slowdown = 3.8×): storage **0%** (0% iowait, zero reads, no swap); clocks ~**1.1×** (5.7→5.16 GHz all-core); GC stop-the-world pauses ~**1.04×** (79→214 ms/game, ~1.4% of game time); residual **~3.3× = mutator memory stalls** — L3/DRAM contention from ~575 MB/s/worker allocation traffic. # Session 3 (night): M0 design phase closed — override plan, bridge protocol, harness spec

## Done

- **`PlayerController` override plan** ([playercontroller-override-plan.md](../design/playercontroller-override-plan.md)): three parallel archaeology agents over the fork, load-bearing claims verified against source. Injection is zero-engine-change (`LobbyPlayerAnvil` via `IGameEntitiesFactory`); surface is 110 abstract methods (~88 real decisions; the incumbent AI stubs/randomizes ~a dozen). **Decision (user):** `PlayerControllerAnvil extends PlayerControllerAi` — inherits a legal answer for every method day one plus the AI cast machinery — with hard per-decision provenance tagging (`answeredBy: bridge | heuristic-fallback`) so fallback never contaminates training truth.
- **Bridge protocol v0** ([bridge-protocol-v0.md](../design/bridge-protocol-v0.md)): wire schema is six game-agnostic *answer shapes* (SELECT_ONE/K, INT_IN_RANGE, BOOL, ORDER_N, CONSTRUCT); Magic semantics are namespaced tags + opaque context — §1 hygiene realized at the wire. One bidi gRPC stream per worker; server-side `bridged_tags` drives coverage; one-shot cast = composite `CastPlan` behind a config flag (ADR-0001's "retreat is a config change", literally); fallback is a first-class response with alarmed telemetry; trajectories stay worker-side. `yield_directive` field reserved for §3b macro-stops.
- **M0 batch-harness spec** ([m0-batch-harness-spec.md](../design/m0-batch-harness-spec.md)): the **chunk** (200-game worker invocation, JVM exits when done) is one mechanism for recycling, resume, pause, and crash isolation; `run.json` manifest is the formal pinning machinery (fork commit + jar sha256, worker refuses on mismatch); `replay <run> <index>` is a first-class verb; defaults transcribed from ADR-0002.
- Landmine logged for M2: `GameCopier.clonePlayer` silently swaps any non-AI `LobbyPlayer` to `LobbyPlayerAi(USE_SIMULATION)` in the copy — forks of Anvil-controlled games would hand the fork to the heuristic AI. One-line fix, goes with the fork-API work.

## Broke / surprised me

- Best surprise of the archaeology: `playChosenSpellAbility` is *itself* an abstract controller method — the engine hands the whole cast flow back to the controller. Forge already contains both answer shapes ADR-0001 demanded: the human path is serialized micro-step callbacks, the AI path is one-shot (pre-set targets + `AiCostDecision`). The one-shot-or-micro-step invariant maps 1:1 onto existing engine structure; no impedance mismatch to bridge.
- One claim in the granularity analysis is inferred, not observed: the exact mode/X/optional-cost callback order on the AI cast path. The instrumented callback census is the empirical check, and it's deliberately the first implementation step.
