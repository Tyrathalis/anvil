## Broke / surprised me

- The "stuck" model server was Python block-buffering stdout to a file —
  the process was loaded and listening the whole time. A port probe beats
  staring at a silent log.
- First smoke run forgot `--census` (the flag exists per-run; --obs alone
  doesn't imply it). The standing rule says census+obs for eval runs —
  cheap re-run, but worth remembering the flags are independent.
- Player-index conventions differ per head family (target decoder:
  absolute registered index; combat heads: self-first position). Both are
  internally consistent train↔serve, but the serve translation is where a
  mix-up would have been silent. Caught by reading, not by a test — the
  combat parity test now pins the position convention.
