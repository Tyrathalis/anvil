- **Build plan pinned: [d6-vtrace-loop.md](../design/d6-vtrace-loop.md).**
  Headlines: synchronous iterations (alternation costs <15% wall-clock, buys
  zero GPU contention + no hot-swap machinery + per-iteration provenance);
  **action = the model's sampled composite pick, realizer+engine =
  environment** (IS ratios exact by construction; vetoes/drops are
  environment stochasticity); behavior policy recorded server-side to
  `mu.jsonl` keyed (g, s) — the dec seq the store already joins dec/ret on —
  zero Java changes; reward terminal-only, win=1, **draw/cap/crash≠win for
  both seats** (§3d); V-trace ρ̄=c̄=1; replay over last 4 iteration stores;
  critic = the policy's own masked value head (full-vis critic = documented
  upgrade, stays the Ante/eval critic).
- **Sampling path (anvil/policy/sampling.py + act() + server):** Gumbel-max
  per head with per-decision Philox noise seeded by splitmix64(game_seed, s)
  — generated at the ITEM's unpadded shapes and scatter-padded into the GPU
  micro-batch, so samples are deterministic per decision and independent of
  batch composition (twin-determinism survives batching; contiguous padding
  would have silently misaligned the player/STOP/none columns). `act(noise=)`
  returns fp32 per-head logp/entropy; noise=None is byte-identical to the
  greedy path. Server `--sample --temperature --mu-out`; `mu_record` writes
  actions in item-canonical index spaces.
- **Learner (anvil/training/rl.py):** `composite_logp` = sum over LABELED
  factors — inclusion rules live in the loader's label construction
  (`apply_mu_labels`, lockstep with `mu_record`); one body serves the mu
  recompute tripwire, the π ratios, and the differentiable PG term.
  V-trace targets per seat-trajectory (unit-tested: on-policy ⇒ Monte-Carlo,
  calibrated ⇒ zero advantage, clipping bounds corrections; caught a real
  double-counting bug — terminal value must be 0, reward rides the last
  transition). Trajectory streaming via the serve featurizer path
  (`store_wire_hist` promoted from the parity test) — loader windows are
  serve-identical by construction. Two-pass per trajectory: no-grad values +
  logp_pi → targets; grad segments ≤256 → PG + value BCE(soft vs) + entropy
  bonus.
- **Ingest/store:** `mu.jsonl` merged like rollout labels (last-wins on
  (g,s) — re-issued crashed games answer again; mismatches caught by the
  tripwire); `TrajectoryStore.mu_for_game()` lazy reader; MultiStore
  delegates.
- **Loop driver (anvil/training/selfplay.py):** serve ckpt_k sampled →
  generate (disjoint --start-index per iteration, one seed stream) → ingest →
  V-trace train on replay mixture → ckpt_{k+1} → monitor.jsonl row (census
  fallback/veto/clamp, statuses, turns; rl reward-vs-critic anomaly rule §6,
  tripwire count, ρ/KL/entropy) → restart server. STOP-file graceful exit +
  loop_state.json resume. Arms every N iterations (argmax serve, paired
  seeds, both seat assignments, arms_report.py).

## Broke / surprised me

- **V-trace terminal bootstrap double-count**: first cut put the reward in
  both r[-1] and V(x_T); on-policy targets came out 2.0. Caught by the
  reduces-to-Monte-Carlo unit test before any training ran.
- **py3.14 forkserver strikes again**: `collate_fn=lambda` isn't picklable —
  DataLoader workers need module-level functions (same class as the
  no-tree-edits-during-runs rule).
- **The mu tripwire fired at tolerance 0.05 on ~0.5% of decisions** — all
  benign: picks match, |Δlogp| ≤ 0.075, concentrated on soft heads (the x
  head at non-X casts, ent ~0.8–1.6) — bf16 serve (micro-batched) vs
  recompute noise. Tolerance raised to 0.2 (real skew shows pick mismatches
  or O(1)+ deviations). Watch item: the x factor is included for every cast
  (v0 rule) but the x head is untrained on non-X casts — pure ratio variance
  + noise gradient; refine inclusion to X-carrying SAs if ent_x drifts.
- **Integer replay weights multiply data** — fresh×3 + 3 old = 6 store-scans
  ≈ 1.8 h/iteration. Fix in flight: fractional weights (subsample old stores;
  fresh 1.0 + old 0.33 each ≈ 2 store-scans, 50% fresh).

- **End-to-end smoke: THE LOOP IS CLOSED.** 2 iterations × 24 games through
  the full driver (exit 0): iteration 1 served the RL-trained ckpt_1,
  replay-mixed both stores, 0 tripwire violations at the calibrated
  tolerance, KL(π‖μ) ~1e-3 at lr 1e-5, critic v0 0.49–0.52 vs realized 0.5
  (no §6 anomaly), monitor flagged exactly the 1 known block-realizer
  fallback. ~5.5 min/iteration at smoke scale. Fixes en route: py3.14
  lambda-collate pickling; fractional replay weights (integer repeats would
  have made an iteration 6 store-scans ≈ 1.8 h — now fresh 1.0 + old 0.33
  each ≈ 2 scans); monotonic ckpt step numbering across the BC→RL chain (the
  tripwire's mu_step gate needs step to uniquely name the generating ckpt).
  Smoke record: `data/training/d6-smoke/` (+ its runs/stores).

- **Run 1 first contact found a real bug in 25 minutes** (`76341c6`): a
  worker OOM'd mid-game-237 (ExitOnOutOfMemoryError + chunk re-issue worked
  as designed), but BOTH attempts wrote mu records and `s` restarts per
  attempt — the last-wins (g,s) merge built a chimeric action stream where
  the sampled trajectories diverged, and an out-of-bounds candidate label
  crashed the gather kernels at trajectory 320. Fixes: ingest drops ALL mu
  records of any game with conflicting duplicate keys (identical duplicates
  keep — same seeded noise ⇒ same records on a faithful re-run);
  `mu_matches` bounds-validates every record against its rebuilt window in
  the loader as the backstop (skip reason `mu_mismatch`). Verified on the
  crashed run dir: game 237 detected, 1,811 records dropped, 281,467 clean.
  Run relaunched from scratch (same seed base — deterministic regeneration).
