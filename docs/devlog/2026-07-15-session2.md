## What broke / lessons

- **rl.py first-log-row dilution** (pre-existing): the metrics flush divides
  by the nominal window (`traj_per_step × log_every`) but the modulo runs on
  the *absolute* step counter, so the first row after a non-aligned init step
  covers a short window — run-1's first rows read ~2× diluted (rho 0.50,
  reward 0.26). Rediscovered when the smoke's only row looked alarming
  (ρ 0.35, reward 0.175) — the loader probe showed reward 0.5 / ρ 1.0 and
  localized it to accounting. Fixed: normalize by actual trajectories since
  last flush. Run-1 conclusions unaffected (its summaries leaned on later
  healthy rows).
- **Post-D5 latent skew in Ante**: `ledger.py` value-only examples lacked
  the combat fields D5's collate reads per-example (`KeyError: 'cmb_rows'`)
  — this was the first certify since D5 landed. Fixed with the loader's
  empty-combat form.
- **`&`-backgrounded servers ignore SIGINT**: a child of a non-interactive
  shell inherits SIGINT=SIG_IGN and Python leaves it ignored, so the
  re-baseline script's `kill -INT` + `wait` stalled ~45 min after the d5
  arms until a status check caught it. SIGTERM works; the selfplay driver
  never hits this because it Popens the server directly.
