## τ-strength curve: the lever is falsified ([§6e resolved](../design/d6-vtrace-loop.md))

- `scripts/tau_curve.py` (formalized): arms at τ ∈ {0.3, 0.5, 0.8, 1.0},
  400 paired games each on run-6 iter-19, same pairs/seeds as all standing
  arms; iter-19's in-run argmax arm (0.5400) = the τ→0 point for free.
  Record `data/runs/tau-curve-run6i19-report.json`.
- **FLAT: 0.5200 / 0.5025 / 0.5200 / 0.5075.** Paired-vs-argmax −2.0 to
  −3.75pp (±~2.1) at every τ; the CRN-sharpened τ-vs-τ reads (shared seeds
  AND shared underlying noise) put τ=0.3 at just **+1.25pp ± 2.0 over
  τ=1.0** (t=0.63). τ=1 point 0.5075 cross-validates run-6's heur-half
  0.5064. Mechanics confirmed live: first-attempt veto rises monotonically
  with τ (0.105→0.128) — behavior moved, outcomes didn't.
- **Interpretation (the reframe that redirected the day):** with entropy
  ~0.17 the policy is peaked; τ<1 already plays argmax where the model is
  confident, so surviving deviations sit in NEAR-TIE windows whose
  probabilities barely move with temperature. The tax is the price of
  exploration itself. The actionable defect is that exploration variance
  never gets credited back to the near-tie decisions that caused it — a
  credit-assignment problem, not a temperature problem. The τ lever and
  the queued full-vis-critic lever converge on the same object.
