# 2026-07-08

## Product-candidate notes (design discussion, §11-era — no near-term action)

Cluster of Mentor/Tutor-era product modes that emerged from discussion; all are
composition of already-planned machinery (skill token + ladder, Ante, value
function, Grindstone provenance, blunder model) — zero new learned components:

- **Progression mode**: per-game player-skill estimate (per-decision quality vs
  value function from the player's info set — ~600 windows/game beats 1 outcome
  bit; Ante strips variance) → Glicko-style update → skill token for the next
  game. Package as *visible* levels ("promoted to 7"), not invisible
  rubber-banding. **Tuned-fight mode** (pick target winrate): same loop + Elo-gap
  →winrate mapping from the frozen ladder; difficulty must move via the skill
  token (human-shaped play, Maia lesson), never blunder injection — Ante's
  sandbag detector would flag the model's own throws otherwise.
- **Player rating**: two-tier claim. Tier 1 "Anvil level" (decision-quality
  based, internally consistent, no PvP data needed — chess-accuracy precedent).
  Tier 2 "human-equivalent estimate" only after sparse anchors exist (pro games
  at the high end, opt-in external-rating linkage, small PvP set) — never claim
  human Elo from bot games without anchors.
- **Deck assessment beyond tiering**: performance curve = winrate(skill token)
  vs reference gauntlet (slope = skill-dependence, intercept = forgiveness,
  asymptote = ceiling). Decomposes: decision density (the honest-metric
  machinery already counts real decisions/game), decision criticality (top-2
  EV-margin distribution), punishment profile (blunder model × value function),
  consistency (Ante ledger aggregated per deck). Crude versions extractable
  from the current corpus (per-deck decision density, outcome entropy, the
  heuristic's per-deck timeout rate as a board-complexity proxy). Tutor upgrade:
  "best deck *for your level*."
- **Player-facing drills** (user insight): Grindstone drills are already
  playable artifacts (forked real state, seeded, engine-adjudicated) — open the
  interface to players as a practice system. Provenance invariant ⇒ drills from
  *your own* games (Mentor flags the value-trajectory crater, you replay until
  you find the line; value function grades attempts, rollouts strip draw luck).
  Solve-rate-by-skill-token = auto difficulty rating; scenarios are seed+commit
  = shareable daily puzzles; info-set machinery keeps replays leak-free.
  Upstream angle: a practice mode is a feature Forge's player community wants —
  strengthens the contribution story beyond "better bot," and rides the
  flagship fork-API work.

Follow-on refinements (same discussion, drills as first-class Forge feature):

- **Opt-in placement solves the recording-flywheel cold start**: the data-
  collection consent lives behind the drill-mode button, where it is the
  feature's own on-switch ("drills from your games" requires your games) —
  consent-as-enablement, not consent-as-favor. Front-page placement earned on
  feature merit, not begged for the bot's sake.
- **Two puzzle modes = two measurement surfaces**: attempts mode (retries;
  time/attempts-to-solve) is the practice surface and yields rating-binned
  human attempt data (Maia fine-tune fodder); hard mode (one try) is the
  calibration surface (first-attempt success = clean difficulty/rating signal).
- **Duplicate-Magic dailies**: seeded determinism gives every player the
  identical shuffle — luck-free leaderboards (duplicate bridge for Magic), a
  format paper Magic structurally can't offer.
- **Tiered shipping**: tier-1 drills need NO model (engine-adjudicated
  objectives: find-lethal, survive-the-turn — fork machinery + UI only);
  EV-graded attempts and auto-difficulty arrive with the value function and
  skill ladder. The community feature can lead the ML story, and the opt-in
  surface starts collecting before M2 machinery ships.
