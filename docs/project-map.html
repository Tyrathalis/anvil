How the pieces connect

Two invariant boundaries: the engine adjudicates every claim a learned component makes (the realizer enforces legality, never heuristic judgment), and the information set is enforced in exactly one leak-tested Python transform — the same featurization code path serves training and play (tests/test_serve_parity.py).

M2 — Reinforcement learning open (plan: m2-rl-plan.md, seeded from ADR-0010)
  • 1. Fork-API hardening in progress — stable single-step fork API (rollout prerequisite; the flagship upstream contribution, per ADR-0002). The forkcheck harness is the standing regression gate. Day one (07-10): static corruption 12%→0 (face-down exile leak + effect-card field wiring); trajectory divergence 50%→13.4% — root cause was GameCopier renumbering card ids (~10-line fix; residual = this-turn copy gaps, 67 repro seeds); twin determinism 99% (new -twin gate: two copies + same RNG snapshot must replay identically; the 1% tail = hash-ordered iteration in decision paths, isolated with the -bridge stress amplifier — bounds bit-exact replay only, doesn't bias rollout labels). Baselines: data/forkcheck/run-20260710{,-twin}/. Remaining: fork-side Obs re-binding → model-policy rollout smoke + throughput (prices step 4) → upstream PR #1.
  • 2. SA-level action schema + BC retrain — the D8 order rung guessed 31–36% of casts (2.5× the corpus estimate): the largest known winrate leak, and RL needs SA-level action identity anyway. Partial labels from the string join now, exact oi labels accumulating; loader history fix rides along. Entity representation untouched (cards stay single entities).
  • 3. Winrate arms reprice (~12 min each) — attribute the schema change before RL confounds it.
  • 4. Rollout-labeled value targets under the BC policy — the measured fix for the value head (BC label = hidden-info entropy floor ~0.57 BCE); rollouts by the near-parity agent, not the heuristic. Critic goes asymmetric (§4, full-visibility input); the Ante mirror certification (§7/§13) rides behind it.
  • 5. RL fine-tune — V-trace, §6 phase 2, from the SA-level checkpoint; start action-rich (δ=0), let reward set the pass boundary.
  • Combat constructs (D9) — explicit slot after the schema work (fully heuristic at play; labels already in corpus; no telemetry yet naming it the bottleneck). Then mode/allocation heads (1.2% / 0.03% mass). Carried context: yield macros (§3b), ID dropout = the pre-Tutor lever.

Design invariants (standing law)

  • The engine adjudicates every claim any learned component makes.
  • Every LLM judgment is downstream-verified; LLMs never generate training truth.
  • Every drill is provenance-traced to a real game.
  • The value function is continuously audited against rollouts.
  • Detection is the engine's job; response is the model's.
  • The error-accounting queue is the spine everything reports to.
  • The model never sees the engine version — formats are rules to play to; versions are bugs not to learn.
  • Seed everything; deterministic replay. Provenance on all trajectories. Pinned fork per run.
