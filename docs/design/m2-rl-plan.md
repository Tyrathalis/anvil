# M2 plan — RL from the BC start

**Date:** 2026-07-10. **Anchors:** design §3 (action decomposition), §4 (value/critic), §6 phase 2 (V-trace), §7 (Ante), §9 (state forking); [ADR-0009](../decisions/ADR-0009-m1-closeout.md) (M1 closeout, entry points); [ADR-0010](../decisions/ADR-0010-m2-opening-sequence.md) (opening sequence — this plan's seed); [ADR-0002](../decisions/ADR-0002-fork-api-gate-resolution.md) (fork fidelity characterization, `forkcheck`); [ADR-0005](../decisions/ADR-0005-d3-label-mask-semantics.md) (candidate/label semantics); [m1-bc-plan.md](m1-bc-plan.md) (the pattern this doc follows).
**Question answered:** what M2 builds, in what order, and what closes it — such that the RL loop inherits an action space that knows which action it took, a critic that beats the matchup-entropy floor, and forking machinery trusted enough to label value now and power search later.

## The milestone in one paragraph

M1 closed with a pure imitator at 46.8% ± 2.5pp against its own teacher on held-out matchups, with inference in the loop operationally free ([ADR-0009](../decisions/ADR-0009-m1-closeout.md)). M2 turns the imitator into a learner, in ADR-0010's order: harden the fork API (the rollout prerequisite and the flagship upstream contribution), move the action interface to SA-level candidates (the measured 31–36% order-rung leak — the largest known winrate leak — and the action identity RL needs regardless), reprice on the standing 12-minute arms so the schema change is attributed before RL confounds it, replace outcome value labels (measured at the matchup-entropy floor, 2026-07-06 diagnosis) with BC-policy rollout labels feeding an asymmetric critic — which the Ante mirror certification then validates (§13 puts the certification test in M2) — and run the first hand-rolled V-trace self-play from the SA-level checkpoint at δ=0. The close is machinery-shaped per §13: the loop runs and measurably improves; "clearly beats the heuristic" (§15 row 1) is the number the loop grinds toward, recorded per checkpoint, not a precondition for closing.

## Settled decisions (ADR-0009/0010 + carried caveats, consequences here)

1. **The order is ADR-0010's:** fork hardening → SA schema + BC retrain → arms reprice → rollout value labels under the BC policy → RL fine-tune. Avoids retraining twice (the RL init already speaks RL's action space) and cleanly attributes the schema delta.
2. **SA-level candidates change the action interface only.** Candidates become (host entity, SA-descriptor) pairs; the entity/state representation is untouched — cards stay single entities (Tutor-relevant identity preserved). Labels: the sa-string join resolves ~69% of the existing corpus's multi-SA cases (ambiguous cases mask out of the SA-level loss); exact `oi` labels ride every run since 2026-07-10. **No corpus regeneration.**
3. **The loader nested-window history fix** (parent rets joined from the future) rides the D2 retrain — one retrain pays for both.
4. **Rollout value labels come from the BC agent, not the heuristic** (~1,000 g/h, near-parity, on-policy for the policy being improved — ADR-0010 amending ADR-0009). The critic goes asymmetric per §4 (full-visibility input; the obs log already carries full state — the info-set transform stays on the policy tower only).
5. **RL init: the SA-level checkpoint at δ=0** (action-rich; the reward decides the pass boundary). V-trace/IMPALA hand-rolled per §6 phase 2 — not PPO (chronically off-policy mixtures), not MuZero (real simulator available).
6. **Winrate arms (~12 min) are the standing per-checkpoint strength eval.** Census + obs stay on for all eval runs; the standing protocol rule holds: never read a bridge answer field without checking `fallback` first.
7. **Mode heads (1.2% of casts) and allocation heads (0.03%) stay queued behind SA-level candidates**; the modal mode-less cast and divided even-split stay interim executor approximations until then. *(Amended by ADR-0012 / decision #12: these are RL-expressiveness items — do not expect or gate on BC winrate from them.)*
8. **Combat constructs are a hard prerequisite for D6, not a flexible slot** *(amended 2026-07-11, ADR-0012)*. The original telemetry-directed framing used the mass-implies-leak inference D3 falsified: under BC, combat constructs will predictably reprice as a winrate tie (they mimic the same heuristic), so no telemetry will ever summon them — yet RL cannot learn combat it cannot express, and combat is a large share of win-relevant decisions. D5 is sequenced before D6 on expressiveness grounds alone.
9. **The fork stays pinned through hardening.** An upstream rebase (anchor: #11161) is a separate, later dataset-boundary event with `forkcheck` as the regression gate — hardening work and version bumps don't share a change.
10. **Yield macros (§3b) stay parked.** ADR-0004 moved them to M2 for RL-era reasons (stop-setting needs reward, not pseudo-labels), but they are not on the critical path: revisit after the first V-trace loop runs, when episode wall-clock and held-priority skill are measurable. The `yield_directive` protocol hook stays reserved.
11. **Label resolvability, measured (ADR-0012 correction to decision #2):** the sa-string join + identical-key candidate collapse resolves **99.92% exact + 0.08% prefix; masked = 0.004%**, not the ~69%/31% priced above — the "ambiguous" class was identical-string duplicates that collapse into one candidate. Recorded here so the plan's own decision #2 doesn't mislead.
12. **The BC stage is finished as a strength program (2026-07-11, ADR-0012 addendum).** Imitation is at its teacher-clone ceiling with mechanically clean execution; every remaining executor-polish item will reprice as a tie for the structural reason D3 measured (the executor's tie-breaks approximate the same heuristic the policy imitates). From here to D6, pre-RL work is justified ONLY by: **(a) expressiveness RL needs** (D5 combat; mode heads if/when indicted), **(b) label/observation correctness** (D4 machinery), or **(c) eval sharpness** (Ante certification) — never by projected BC winrate. Winrate arms demote from goal to instrument: regression tripwire before RL, progress meter during it. The veto class (~10% of cast attempts, ADR-0012) explicitly waits for RL telemetry — both calibration arms already showed veto reduction buys no BC winrate.

## Deliverables, in order

**D1 — Fork-API hardening.** The contract rollouts need is **state fidelity + determinism, not trajectory reproduction**: ADR-0002's 50% trajectory divergence is *not* a defect for rollout labeling (a rollout is a new game from the forked state, not a replay); the defects are the silent state corruptions and nondeterministic construction. Scope: (a) fix the two seed-reproducible static bug classes (face-down exiled cards flipping face-up in the copy — an information leak a rollout could exploit; the compounding Keyword-Effects duplication) — these are upstream PR #1 per ADR-0002, small and differential-tested, the ~60-seed repro set as the test suite; (b) fix the `GameCopier.clonePlayer` landmine (non-AI controllers swapped to heuristic AI on fork — forked games must keep the Anvil controller or BC-policy rollouts silently become heuristic rollouts); (c) deterministic copy-construction ordering + RNG state capture/restore (the two evidenced API requirements); (d) extend `forkcheck` to gate the rollout contract: fork mid-game → digest-identical state → deterministic seeded completion **under the bridge**. Measure fork+rollout throughput (fork-at-decision, play to end under the BC policy) — this number prices D4. Done: forkcheck clean on the contract; a mid-game fork completes reproducibly under the model; PR #1 out.

**D2 — SA-level action schema + BC retrain.** The pointer head scores candidate rows instead of host entities (PASS stays candidate 0; ADR-0005's candidate rows already carry `{"e","sa","kind"}`). In-deliverable design choice: SA-descriptor featurization — learned embedding over the pinned 105-method vocab + kind/cost features vs frozen-text-embedding the SA string; decide with a cheap ablation if the first attempt disappoints. Labels: exact `oi` where present, sa-string join elsewhere (~69% of multi-SA cases), ambiguous mass masked from the SA-level loss and **reported** (the masked share is measured, not assumed ignorable). Executor: the model now names the SA — the order rung is deleted; the ladder shrinks to shape→pay; rung/veto telemetry stays. Loader rider: the nested-window history fix. Retrain on the known recipe (3 epochs, ~14 h nice-19, 113,592-game corpus); report per-tag honest agreement on **both bases** (host-level for continuity with 0.9758, SA-level as the new headline); re-run pass calibration (fresh δ table beside the checkpoint).

**D3 — Arms reprice.** Same harness, same 200 held-out valpair pairs mirrored, 1,000 games, δ=0 + rate-matched δ arms. The delta vs 46.8% prices the schema change; the multi-SA order-rung telemetry should read ~0. ~12 min/arm — from here, every M2 checkpoint gets this eval.

**D4 — Ante certification, then rollout-labeled critic.** *(Order flipped 2026-07-11, ADR-0012: with every pre-RL lever reading "tie at ±2.5pp," eval resolution is the binding constraint — certify the variance-reduced arm FIRST so the D4 pilot's own go/no-go reads and every subsequent checkpoint arm benefit.)* First the **Ante certification test** (§7, an afternoon): identical-deck mirror batch — ledger sums to zero in expectation, AIVAT-corrected winrate converges faster than raw; §13 explicitly puts this in M2. Then the critic machinery: position sampler over the store (stratified by turn/phase, provenance-tagged), fork-at-decision via D1, K rollouts under the SA-level BC policy on both seats, label = outcome mean. **Pilot before pricing:** measure positions/hour end-to-end and whether a value fine-tune on a pilot-sized label set beats the measured ~0.60 matchup-prior BCE floor and lifts final-turn AUC off 0.68 — the 2026-07-06 diagnosis says label noise is the binding problem, the pilot confirms it before a multi-day labeling run. Decide value-head-only fine-tune vs full retrain on the pilot's evidence. Critic input goes full-visibility (§4).

**D5 — Combat constructs (hard prerequisite for D6).** AttackMap/BlockMap heads + realizers; labels already in the corpus since D2-of-M1. *(Reframed 2026-07-11, ADR-0012 / decision #12: sequenced before D6 on RL-expressiveness grounds — its BC arms will predictably tie and that is not a failure signal; the deliverable is that the V-trace loop can express and therefore learn combat.)*

**D6 — First V-trace self-play.** Hand-rolled (~2K lines, §6 phase 2): actors = harness chunks in serve mode streaming trajectories into the provenance store; learner on the 4090; V-trace off-policy corrections + replay mixing; reward = terminal win only, **cap-aware** (a stalling leader must not profit from draws, §3d); model-side concession stays disabled (the §3d gate — early training is exactly the self-sealing-error regime). Init: D2 checkpoint, δ=0. Monitoring from day one: the anomaly monitor (winrate exceeding critic prediction = bug report until proven otherwise, §6), fallback/veto provenance, per-checkpoint arms. Throughput arithmetic, stated up front: ~1,000 g/h ≈ 24K games/day at w=8 — the first RL runs are measured in tens-to-hundreds of K games, not §14's 1–3M budget; the deliverable is a loop that runs, doesn't collapse, and improves the arms, with Grindstone position economy as the M3-era scale lever.

## Risks and open questions

- **Fork hardening depth.** ADR-0002 characterized two static classes at MAIN1-priority forks on precons; the rollout contract exercises new surface (mid-stack state, trigger contexts, the controller swap, DC pool breadth). Budget archaeology; `forkcheck` grows cases as bugs fall. The 2.4%/fork fresh-drift floor is acceptable for single-fork rollout labeling; repeated-fork designs (search) budget for it later.
- **The 69% join is not a random subset** — ambiguous multi-SA cases cluster (commander permission-route duplicates, cost variants). The masked mass is reported per-tag in D2; the exact-`oi` share grows with every run and the RL era relabels the distribution anyway.
- **SA-descriptor featurization** could underperform host-level pointing if the descriptor is too thin — the both-bases agreement report in D2 is the tripwire; the text-embedding fallback is the designed escape hatch.
- **Rollout label economics.** K × positions × (fraction-of-game wall) could be days; the D4 pilot prices it before commitment. Relief valves: fewer, better positions (variance-reduced labels beat volume); §6's short-horizon drill deltas are M3-era.
- **Value ceiling honesty.** Rollout labels fix label noise, not hidden information — the floor moves, it doesn't vanish (the belief head is M3-era). Success is beating the matchup-prior floor and reading boards late-game, not oracle calibration.
- **RL stability at small scale.** Strong BC init + V-trace + replay mixing is the conservative configuration; first sign of collapse → lower LR, raise the BC replay mix. Per-checkpoint arms catch strength regressions the loss curves miss.
- **Self-play distribution shift.** Model-vs-model games leave the heuristic corpus distribution — intended (that's the point of RL), but the critic must track it via the store's on-policy data, and eval arms stay vs the *fixed* heuristic so the yardstick doesn't move.

## M2 is done when

1. The fork API passes the rollout contract under `forkcheck` (digest-identical copy, deterministic seeded completion under the bridge, Anvil controller preserved) and the static-fix upstream PR #1 is submitted.
2. The SA-level BC checkpoint exists, agreement reported on both bases, arms repriced — the order rung is retired from the executor.
3. The rollout-labeled critic beats the matchup-prior floor on held-out data (BCE below ~0.60; final-turn AUC materially above 0.68) and the Ante mirror certification passes.
4. A V-trace run from the SA-level init improves the standing arms winrate outside noise across checkpoints, provenance-clean, winrate recorded per checkpoint. "Clearly beats the heuristic" (§15 row 1) closes whenever the loop delivers it — in M2 if the first runs get there, inherited by M3's scale-up if not.
